# PriceAI 批发合作线索提交模板规划

> 日期：2026-07-14
> 状态：产品文案草案，待确认后实施
> 关联模块：批发合作、API 中转、卡网订阅、批发线索后台
> 上位规划：[PriceAI B2B 批发撮合专区产品规划](2026-06-30_priceai-b2b-wholesale-matching-plan.md)
> 核心结论：保留结构化字段，但不要把提交区做成生硬问卷；根据“买方 / 卖方 × API 中转 / 卡网订阅”自动填入一份已经写完整的示例，用户只需替换内容。每个商品或号池的信息必须在同一行内读完。

## 1. 背景与问题

当前批发合作提交区已经区分：

- 我有批量需求。
- 我有源头供给。
- API 中转。
- 卡网 / 订阅渠道。
- 其他源头。

但现有模板仍有三个问题：

1. 字段过于宽泛，例如“想要什么”“预计量”“批发价格”，用户容易只写“一级代理”“Claude”“长期合作”等无法撮合的信息。
2. 同一商品的信息被拆散到“可供给内容、批发价格、稳定供给量、起批门槛”等多个区域，后台需要来回对照才能看懂。
3. API 中转和卡网订阅使用了相近模板，但两类业务的判断口径完全不同。

本轮不是取消结构化字段，也不是改成一段完全自由的自然语言，而是把模板调整为：

```text
固定字段名 + 已填写完整的真实示例
```

用户打开后直接修改示例，而不是面对一组空白问题重新组织语言。

## 2. 产品目标

提交后的线索应让另一方和 PriceAI 运营者快速判断：

- 对方是谁。
- 具体要买或能卖什么。
- 每个商品对应什么来源、号池或交付方式。
- 对应价格是多少。
- 买方预计采购多少，或者卖方多少起批、每月能供应多少。
- 是否支持测试。
- 售后和结算边界是什么。

理想状态下，运营者不需要先私聊追问“你说的 Claude 是什么号池”“Plus 是正价代充还是成品号”“这个价格对应哪条线路”，就能完成第一轮判断。

## 3. 核心信息结构

### 3.1 保留结构化字段，但不做表格化问卷

R0 可以继续使用一个主要内容输入框，在内容中保留固定字段名，例如：

```text
采购身份：
想要什么：
来源要求：
测试要求：
结算方式：
补充说明：
```

区别在于字段后面不是空白，而是已经填写完整的示例。

### 3.2 同一商品的信息必须放在同一行

买方单项结构：

```text
具体模型或商品 + 号池/来源或交付方式 + 心理价位 + 预计量/首次试单
```

卖方单项结构：

```text
具体模型或商品 + 号池/来源或交付方式 + 批发价格 + 起批门槛 + 稳定月供
```

正确示例：

```text
- ChatGPT Plus 官方正价代充：140 元/单，10 单起批，每月稳定供应约 500 单
```

不建议拆成：

```text
可供给内容：ChatGPT Plus 官方正价代充
批发价格：140 元/单
起批门槛：10 单
稳定供给量：每月 500 单
```

### 3.3 商品信息与公共条件分开

以下内容属于每个商品，放在商品行内：

- 模型或商品名称。
- 号池、上游来源或交付方式。
- 价格。
- 买方预计量或卖方起批量、稳定月供。

以下内容属于整条线索，放在商品列表下面统一说明：

- 来源披露要求。
- 稳定性要求。
- 测试方式。
- 售后规则。
- 结算方式。
- 可提供的证明。
- 补充说明。

### 3.4 独立字段不在正文重复

页面已经有独立输入框的内容，不再放进模板正文：

- 联系方式。
- 证明链接。

正文仍可写“可提供的证明”摘要，但具体 URL 使用独立证明链接字段。

## 4. API 中转口径

API 中转不能只写“Claude”“ChatGPT”或“Codex”，至少要同时说明具体模型和号池 / 来源。

### 4.1 模型与号池是两个维度

模型示例：

- Claude Sonnet 5。
- Claude Opus 4.8。
- GPT 5.6 Sol。
- GPT 5.6 Terra。
- GPT 5.6 Luna。
- GPT 5.5。

号池或来源示例：

- 官方 API。
- 云厂商线路。
- Claude Max 官方订阅转 API。
- CCMax 满血号池。
- ChatGPT Plus / Pro / Team / Business 号池。
- Kiro Pro 号池。
- 第三方 IDE 转 API。
- 逆向 / 非公开接口。
- 二级分销。
- 混池或未披露池。

Kiro 属于来源或账号池，不是模型。更准确的表达是：

```text
Kiro Pro 号池提供的 Claude Sonnet 5
```

而不是：

```text
Kiro 模型
```

### 4.2 API 价格必须与具体号池绑定

不建议只写：

```text
Claude 0.6 元/刀，GPT 0.5 元/刀
```

建议写成：

```text
- CCMax 官转 Claude Sonnet 5：0.6 元/刀，每月预计 10 万刀
- Kiro Pro 号池 Claude Sonnet 5：综合倍率 0.08x，每月预计 3 万刀
- ChatGPT Pro 号池 GPT 5.6 Sol / Codex：0.5 元/刀，每月预计 5 万刀
```

这里的示例数字只用于演示填写格式，不代表 PriceAI 当前市场报价。

## 5. 卡网订阅口径

卡网订阅不能只写“ChatGPT Plus”“ChatGPT Pro”或“Claude Max”，必须进一步说明交付类型或商品规格。

### 5.1 ChatGPT / OpenAI 常见拆分

- ChatGPT Plus 官方正价代充。
- ChatGPT Plus 地区价代充。
- ChatGPT Plus 成品号。
- ChatGPT Plus CDK / 卡密。
- ChatGPT Team / Business 团队邀请、母号或成品号。
- ChatGPT Pro 5x 官方代充或代开。
- ChatGPT Pro 20x 官方代充或代开。
- Codex API / 额度类商品。

### 5.2 Claude 常见拆分

- Claude Pro 正价代充。
- Claude Pro 成品号。
- Claude Max 5x 官方代充或代开。
- Claude Max 20x 官方代充或代开。
- Claude 普号或兑换号。

### 5.3 Kiro 常见拆分

- Kiro 普号 / Free。
- Kiro Pro。
- Kiro Pro+。
- Kiro Pro Max。
- Kiro Power。
- Kiro 额度号。

### 5.4 高风险来源边界

以下内容不能作为正常可撮合商品，与正价代充、地区价代充或成品号并列展示：

- 黑充。
- 盗刷或黑卡支付。
- 明确依赖拒付的商品。
- 撞库、盗号或账号共享访问。
- 来源完全不明且要求大额预付的供给。

这些词可以进入风险识别、后台标签或禁入判断，但不应进入正常供货模板示例。

## 6. 四套最终示例模板

以下数字均为填写格式示例，不代表真实市场报价或 PriceAI 推荐价格。

### 6.1 API 中转买方示例

```text
采购身份：企业用户，主要用于 Claude Code 和 Codex

想要什么：
- CCMax 满血号池，Claude Max 官方订阅转出的 Claude Sonnet 5：心理价位 0.6 元/刀，每月预计 10 万刀
- Kiro Pro 号池的 Claude Sonnet 5：心理价位综合倍率 0.08x，每月预计 3 万刀
- ChatGPT Pro 号池的 GPT 5.6 Sol / Codex：心理价位 0.5 元/刀，每月预计 5 万刀

来源要求：需要说明具体号池，不接受未披露混池；更换来源或号池需要提前说明

稳定性要求：要求模型真实，不随意降级，能够长期稳定运行 Claude Code 和 Codex

测试要求：希望先分别小额测试，确认模型和稳定性后再正式采购

结算方式：可以预付，稳定合作后可以沟通其他结算方式

补充说明：希望寻找可以长期稳定供货的上游
```

### 6.2 API 中转卖方示例

```text
源头类型：自建并维护 CCMax、Kiro Pro 和 ChatGPT Pro 号池

可供给内容：
- CCMax 官转 Claude Sonnet 5：0.55 元/刀，首次充值 1000 元起，每月稳定供应约 20 万刀
- Kiro Pro 号池 Claude Sonnet 5：综合倍率 0.07x，首次充值 1000 元起，每月稳定供应约 8 万刀
- ChatGPT Pro 号池 GPT 5.6 Sol / Codex：0.45 元/刀，首次充值 1000 元起，每月稳定供应约 10 万刀

测试方式：每个号池都支持小额付费测试，满意后再正式充值

售后说明：号池调整、模型切换或大面积限流会提前通知，短期故障根据影响情况补偿

结算方式：支持人民币或 USDT 预付，稳定合作后可以沟通其他方式

可提供的证明：号池后台脱敏截图、公开监控和历史供货记录

补充说明：不同号池独立计价，不会用其他低价线路替换
```

### 6.3 卡网订阅买方示例

```text
采购身份：卡网店主，主要销售 ChatGPT、Claude 和 Kiro 商品

想要什么：
- ChatGPT Plus 官方正价代充：心理价位 145 元/单，首次试 10 单，每月预计 100 单
- ChatGPT Plus 地区价代充：心理价位 75 元/单，首次试 20 单，每月预计 300 单
- ChatGPT Pro 5x 官方代充：心理价位 720 元/单，首次试 5 单，每月预计 30 单
- Claude Max 20x 官方代充：心理价位 1450 元/单，首次试 5 单，每月预计 20 单
- Kiro Pro Max 成品号：心理价位 30 元/个，首次试 50 个，每月预计 200 个

商品要求：代充需要说明正价、地区价或其他来源；成品号需要说明地区、额度、能否换绑和有效期

售后要求：代充失败可以退款，成品号需要提供首登保障，短期失效按约定补单

结算方式：前期按批预付，合作稳定后可以沟通其他方式

补充说明：不同来源和交付方式请分别报价
```

### 6.4 卡网订阅卖方示例

```text
源头类型：AI 订阅代充和成品号一手供货渠道

可供给内容：
- ChatGPT Plus 官方正价代充：140 元/单，10 单起批，每月稳定供应约 500 单
- ChatGPT Plus 地区价代充：70 元/单，20 单起批，每月稳定供应约 300 单
- ChatGPT Pro 5x 官方代充：700 元/单，5 单起批，每月稳定供应约 50 单
- Claude Max 20x 官方代充：1400 元/单，5 单起批，每月稳定供应约 30 单
- Kiro Pro Max 成品号：25 元/个，50 个起批，每月稳定供应约 1000 个

交付说明：代充商品正常在 30 分钟内处理，成品号批量发货，需要提前确认库存

售后说明：代充失败可以退款，成品号提供 24 小时首登保障，具体失效处理规则采购前说明

结算方式：按批预付，支持人民币和 USDT

可提供的证明：卡网链接、脱敏订单、商品截图和历史供货记录

补充说明：不同商品、来源和交付方式分别报价，不提供来源不明的商品
```

## 7. 页面行为建议

### 7.1 根据两个选择自动切换模板

模板由两个条件共同决定：

| 角色 | 方向 | 使用模板 |
| --- | --- | --- |
| 买方需求 | API 中转 | API 中转买方示例 |
| 卖方供给 | API 中转 | API 中转卖方示例 |
| 买方需求 | 卡网 / 订阅渠道 | 卡网订阅买方示例 |
| 卖方供给 | 卡网 / 订阅渠道 | 卡网订阅卖方示例 |
| 买方或卖方 | 其他源头 | 使用精简通用模板 |

### 7.2 模板切换不能覆盖已填写内容

- 用户尚未修改默认示例时，切换角色或方向可以自动替换模板。
- 用户已经修改内容时，不应静默覆盖。
- 如需切换，可以提示用户确认是否替换为新的示例。

### 7.3 示例不是占位符

示例作为可编辑正文展示，让用户直接改写；但提交时不能只依赖输入框非空判断。

至少需要确认：

- 用户修改过默认示例，或者明确确认示例就是自己的真实情况。
- 至少保留一条具体商品或号池信息。
- 买方条目包含心理价位和预计量。
- 卖方条目包含报价、起批门槛和稳定月供。

### 7.4 R0 不必立即拆成大量独立输入框

本轮“结构化”首先指内容表达有固定字段和固定顺序，不要求立即把所有内容拆成十几个表单控件。

R0 可以保留：

- 标题。
- 方向。
- 一个带完整示例的主要内容输入框。
- 联系方式。
- 证明链接。

后续真实线索量足够，再根据运营筛选需求决定是否把商品条目做成可增删的独立结构化组件。

## 8. 后台阅读与筛选建议

后台首先需要快速看到：

- 买方还是卖方。
- API 中转还是卡网订阅。
- 第一条或前两条具体商品 / 号池。
- 对应价格。
- 对应预计量、起批量或月供量。
- 联系方式。
- 当前处理状态。

后台摘要不应把整份模板压缩成一段连续文字。优先保留商品行的换行结构，并将通用测试、售后、结算信息放进详情面板。

## 9. 风险边界

本模板只帮助提高线索资料完整度，不改变 PriceAI 的产品边界：

- PriceAI 只做线索记录、资料核验和人工撮合。
- 不托管资金。
- 不做平台内付款。
- 不担保交易结果。
- 不保证任何号池、商品或上游长期稳定。
- 不公开或交易 API Key、Cookie、session、token 等敏感凭据。
- 黑充、盗刷、撞库、拒付和其他明确黑产相关供给不进入正常撮合。

涉及订阅转 API、第三方 IDE 转 API、逆向或混池时，应明确披露来源类别和已知风险，不能用“官方”“满血”“一手”等绝对说法替代证据。

## 10. 成功标准

模板上线后，应观察：

1. 多数线索是否能明确对应到具体商品、模型和号池。
2. 买方是否能写清价格和预计量。
3. 卖方是否能写清价格、起批量和稳定月供。
4. 运营者是否能在 30 秒内判断一条线索是否值得跟进。
5. 后续私聊中反复追问“什么来源、什么价、多少量”的次数是否下降。
6. 风险来源是否被主动披露，而不是藏在模糊商品名称后面。

## 11. 当前决策

本轮已经确认：

- 保留结构化字段。
- 字段后面放完整示例，不放一组空白横线。
- 买方的“想要什么”按商品或号池逐行填写，并在同一行写价格和预计量。
- 卖方的“可供给内容”按商品或号池逐行填写，并在同一行写价格、起批量和稳定月供。
- API 中转与卡网订阅使用不同模板。
- API 中转必须区分模型与号池 / 来源。
- 卡网订阅必须区分正价代充、地区价代充、成品号、CDK、团队邀请等交付类型。
- 联系方式和证明链接继续使用独立字段。
- 黑充、盗刷、拒付等只作为风险或禁入类型处理，不作为正常撮合商品。
